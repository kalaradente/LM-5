## Wiring summary

1. **K-MC1 pin strip** (the build's only soldering — 4 wires): VCC → battery 5V ·
   GND → shared ground · **AC-output** IF I → TRS tip · IF Q → TRS ring
   (sleeve = ground) — AC, not DC, per LM's wiring decision (see
   openflight_iwr6843/README.md). Check the datasheet's unused-pin guidance
   for the VCO/FM input before soldering.
2. **TRS cable** (ferrite clipped near HAT end) → HiFiBerry 3.5mm line-in.
   No impedance-matching needed: K-MC1 output is 100 ohm, DAC2 ADC Pro input
   is 20kOhm/pin (~200:1, verified against both datasheets).
3. **HiFiBerry** → stacked on Pi 5 GPIO header.
4. **IWR6843ISK** → one micro-USB to the Pi (carries power + both serial ports).

Both radar boards mount vertically on standoffs, antennas facing the tee,
coplanar and aimed identically. Plate sits on rubber feet 1–1.5m behind the
ball on the target line, front edge shimmed 10° up (matches
`IWR6843Source.MOUNT_TILT_DEG` in code -- if you build a different tilt,
change that constant to match, or launch angle will be systematically
wrong by roughly the tilt-vs-constant mismatch), verified level side-to-side.

## Software (all free)

- TI mmWave SDK + Demo Visualizer (bring-up rung 1)
- OpenFlight (server, physics, UI): https://github.com/jewbetcha/openflight
  (vendored locally at ./openflight_upstream for shot_simulator.py to import)
- The `openflight_iwr6843` package from this design: TLV parser, Kalman trackers,
  spin decoder, session modes, validation harness
- `shot_simulator.py` (repo root): plaintext ball speed/spin/launch angle ->
  full RK4 physics flight sim, optionally pushed live into a running
  OpenFlight server. See its docstring and openflight_iwr6843/README.md.
